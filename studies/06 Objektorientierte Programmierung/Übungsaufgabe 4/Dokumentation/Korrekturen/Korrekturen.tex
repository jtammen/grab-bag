\section{Speicherung des Huffman-Baums}
Um den Huffman-Baum beim Decodieren einer Datei zu rekonstruieren, müssen in der codierten Datei
ebenfalls die Häufigkeitsverteilungen der im Text vorkommenden Zeichen abgespeichert werden.

Um dies möglichst speicherplatzsparend zu erreichen, muss zunächst ein Weg gefunden werden, mit dem
sich \textsl{bit}weise in eine Datei schreiben bzw. aus ihr lesen lässt. C++ bietet standardmäßig
nur einen \textsl{byte}weisen Zugriff. Würde man die Daten byteweise ablegen, würde für jedes
Zeichen max. 3 Byte (ausgehend vom ASCII-Code) verwendet; für die Häufigkeit entsprechend 
$\text{"`Länge der Häufigkeitszahl"'} * 1\; \text{Byte}$.

Für die Speicherung des Bitcodes wird dies noch deutlicher: würde man jedes "`Bit"' des Codes als ein \texttt{char} (1 Byte) speichern, wäre die codierte Datei schnell größer als die Quelldatei.

\subsection{Speicherstruktur}
Mit dem bitweisen Dateizugriff lässt sich etwa folgende Speicherstruktur für die codierte Datei einsetzen.

\begin{figure}[H]
	\centering
		\input images/speicher.pst
	\caption{Beispiel Speicherstruktur}
	\label{fig:codeaufbau}
\end{figure}

Dabei gliedert sich die Struktur in die folgenden Teile: (es wird bereits das Speichern des Bitcodes am Ende
der Datei vorgesehen)

\begin{enumerate}
	\item Anzahl der am Ende der Datei zu ignorierenden Bits. Diese Angabe wird benötigt, da die Bitanzahl stets eine 
	ganzzahliges Vielfaches von 8 ist -- im Huffman-Code kann der Code allerdings auch eine andere Länge haben. Ist dies
	der Fall, wird das Ende der Datei mit Nullen aufgefüllt und beim Decodieren die entsprechenden Anzahl an Bits
	ignoriert. (\textbf{1 Byte})
	\item Anzahl der nachfolgend codierten Zeichen (plus deren Häufigkeiten). Durch diese Angabe lässt sich später
	das Ende der Zeichenhäufigkeiten und der Beginn der codierten Datei ermitteln. (\textbf{1 Byte})
	\item Ab hier beginnt der "`dynamische"' Teil der Datei. Es folgt jeweils zuerst das codierte ASCII-Zeichen
	(\textbf{1 Byte}), anschließend folgt
	\item die dazugehörige Häufigkeit. (\textbf{4 Byte}, entspricht Datentyp \texttt{int})
	\item Nun folgt der nächste Block aus Zeichen und
	\item Häufigkeit
\end{enumerate}

Damit lässt sich der Speicherbedarf für die Ablage des Huffman-Baums wie folgt angeben:

$$1\;\text{Byte} + (\text{Anzahl Zeichen} * 5\;\text{Byte})$$

\section{Text codieren}
Durch Traversieren des Baumes erhält man die Bitcodes der einzelnen Zeichen.

\subsection{Pseudocode}
\begin{lstlisting}[language=Pseudo,caption={Pseudocode Traversieren}]
function traversiereBaum (KnotenZeiger p, Stack s)
if <p != 0>
	if <p ist AussenKnoten>      /// Endknoten ist erreicht
		code = <Inhalt von s>
		s.pop()                  /// Letztes "Bit" entfernen
		codeTabelle.insert(<Zeichen von p>, code)
	else                    /// Kein Endknoten, weiter traversieren
		s.push_back(0)      /// prefix: gehe links: 0
		traversiereBaum( <linkes Kind von p>, s )
		
		s.push_back(1)      /// infix; gehe rechts: 0
		
		traversiereBaum( <rechtes Kind von p>, s )
		s.pop_back()        /// postfix; eine Ebene zurueckgehen
	end if
end if
end function
\end{lstlisting}


\subsection{Speicherung der codierten Daten}
Ausgehend von der obigen Annahme, dass ein bitweiser Zugriff auf die Ausgabedatei besteht, können nun beim Codieren
einer Datei die Bitcodes für die einzelnen Zeichen speichersparend ausgegeben werden. Der Bitcode-String wird
dabei jeweils Zeichen für Zeichen durchlaufen und die einzelnen "`Bits"' in einen Ausgabepuffer geschrieben.

Sind dort nun 8 Zeichen abgelegt, wird das Byte mittels eines Filestream-Objekts in die Ausgabedatei geschrieben.
Wie oben beschrieben, müssen am Ende des Codiervorgangs eventuell "`fehlende"' Bits durch Nullen ergänzt werden;
diese können dann beim Decodieren einfach ignoriert werden.

Der Bitcode \texttt{0010001110011011} z.B. würde nun wie folgt direkt im Anschluss an die Häufigkeitsverteilung in
die Datei geschrieben werden:

	\caption{Beispiel Speicherstruktur, codierte Daten}
	\label{fig:codeaufbau2}
\end{figure}


\section{Text decodieren}
Hierbei muss zunächst aus der Quelldatei der Block am Anfang mit der Häufgikeitsverteilung gelesen und der Huffman-Baum
rekonstruiert werden. Anschließend steht der Dateizeiger an der Stelle, an welcher der Bitcode beginnt. Nun wird Bit für Bit eingelesen und der Baum nach dem entsprechenden Zeichen durchsucht.
